
REGLAS RUTAS/LLAMADAS HTTP

1. Cada llamada http que necesite autenticacion requiere que se envie
en el body tres elementos: [userIdRequest], [token] y [userAction]. 

userIdRequest se refiere al id del usuario que hace la request; el Token al token de la
sesion actual activa que tiene el usuario;  Esto es para que el middleware
'hasValidToken' funcione correctamente.
Accion se refiere a la accion del usuario, esta se guardaen la tabla UserLogs y es
necesario para que el middleware 'logUserAction' funcione correctamente.

2. Si se requiere que en el body se reciba un id diferente al id del 
usuario (userIdRequest) este se debe llamar 'id' simplemente, y si se requieren
enviar más de un id, se usa la convencion '[nombre]Id'. Ejemplo: eliminarId
y crearId.

3. CONVENCION: para los metodos de asociacion (muchos a muchos) se usa
la convencion "asoc-nombre_tabla_a_que_asocia" para la url del metodo. 
EJ: "asoc-user". Con el fin de que la ruta quede así: /tabla1/asoc-tabla2.
EJ: "/rol/asoc-user".
Para los metodos de actualizar o eliminar una asociacion, se usa la
convencion "update-asoc-nombre_tabla" o "delete-asoc-nombre_tabla".
EJ: "/tabla1/update-asoc-tabla2/".

4. Para routers que manejen varios modelos se usa la ruta "/create" y "update" exclusivamente
para el modelo principal de ese router, para el resto se usa "/create-[model secundario]"
